以下に提示される巡回セールスマン問題 (TSP) のインスタンス(地点数、および地点間の距離や位置情報など) をもとに、最短距離のツアーを導出してください。  

# 目的
- 全ての地点 (0 ～ n) を重複なく訪問し、最終的に出発地点に戻るルートを構築すること。  
- 可能なかぎり総移動距離が最小となるよう、最適解またはそれに極めて近いルートを探すこと。  
- 最後に訪問した地点と始点の距離が加算されることに注意する。

# 出力条件
1. 同じ地点を複数回訪問することは不可 (重複禁止)。  
2. 0 から n までの全ての地点をちょうど一度ずつ含むツアーを作成すること。  
3. 見つかった最小総距離 (または最小に近いと推定される距離) を実現するルートを、訪問順にリスト形式で明示すること。  

# 最適に近い解を発見するための戦略
## 全体をグループ化・エリア分割する発想
- 都市の数が多いときは、ざっくりとマップを複数のエリアに分割して考える。  
- ひとつのエリア内はある程度完結したルートを作り、エリア間はなるべく短い経路でつなぐ。  

## 近傍挿入を意識する
- 現在地に最も近い都市へ進むことで一旦の近似解を得る。  
- 全体で見ると最適解とは限らないが、初期解をざっと作るには有効である。

## “交差”を避ける
- 交差する経路は、往復距離やロスが増える傾向にある。
- ルートが交差している場合は、積極的に修正する。

## 最後に訪問した地点と始点の距離を小さくする
- 始点に戻ることを考慮せずにツアーを考えると、最後に大きなコストが発生する。
- ツアーの終盤徐々に始点に近づくように、計画的にツアーを考える。

以上の要件を満たし、かつ最小距離 (または最適に近い距離) のツアーを出力してください。
